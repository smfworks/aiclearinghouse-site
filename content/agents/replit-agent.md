## When to choose Replit Agent

Choose Replit Agent when the entire workflow — coding, testing, deployment, and team collaboration — should live in one browser tab. It is especially strong for education, rapid prototyping, and teams already embedded in Replit workspaces.

## What it does well

- **All-in-one workspace.** Code, dependencies, secrets, databases, hosting, and domains are managed inside Replit. Context switching is minimal.
- **Long autonomous sessions.** Agent 3 sessions can run for up to 200 minutes, making it possible to hand off larger tasks.
- **Subagent delegation.** It can spawn specialized agents for testing, styling, or documentation, dividing labor like a small dev team.
- **Effort modes.** Economy, Power, and Turbo let you trade speed and cost against task complexity.
- **Live collaboration.** Multiple people can view and edit the same Repl while the agent works, which is excellent for teaching and pair building.
- **Built-in hosting.** Deployments to Replit's infrastructure are one click away, with custom domain support.

## Honest limitations

- **Replit ecosystem lock-in.** The easy path keeps everything on Replit. Exporting a mature project to a standard repo takes effort.
- **Variable output quality.** Like many autonomous agents, complex tasks can drift. Direction and checkpoints matter.
- **Pricing at scale.** Free tiers are limited; sustained team use on Turbo or Power modes adds up.
- **No model choice.** You do not pick the underlying model.

## Best fit

Students, educators, indie hackers, and small teams who want a complete browser-based dev environment with an agent that can carry tasks from idea to deployment.
