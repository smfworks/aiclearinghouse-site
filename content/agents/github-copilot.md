## When to choose GitHub Copilot

Use Copilot when your team already lives in GitHub and wants AI assistance with the lowest possible activation energy. It is the default choice for organizations that prioritize rollout speed over maximum capability.

## What it does well

- **Ubiquitous integration.** Copilot works inside VS Code, JetBrains, Vim, Neovim, Visual Studio, and the CLI. It also surfaces in GitHub pull requests and issues.
- **Enterprise governance.** Admin controls, IP indemnity, audit logs, and data residency options make it palatable to security and legal teams.
- **Agent mode.** Recent updates let Copilot execute multi-step coding tasks inside the IDE, closing the gap with Cursor and Claude Code.
- **Billing simplicity.** One subscription covers completions, chat, and agent mode. Procurement already knows GitHub.

## Honest limitations

- **Not the bleeding edge.** Copilot is good at everything, excellent at nothing. Its agent mode lags behind dedicated agents for complex tasks.
- **Microsoft lock-in.** The best experiences are tied to GitHub, Azure, and Microsoft IDEs. Other tools get second-tier support.
- **Limited model choice.** You use the models GitHub selects. There is no BYO-model or local fallback.

## Best fit

Engineering teams in GitHub-centric organizations that need an approved, easy-to-administer AI assistant rather than a best-of-breed power tool.
