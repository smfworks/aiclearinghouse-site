## When to choose PearAI

Use PearAI when you want an AI-native editor but prefer open-source software and the ability to audit or modify the tool. It is a practical alternative to Cursor or Windsurf for privacy-conscious teams.

## What it does well

- **Open-source editor core.** Forked from VS Code, so extensions and keybindings mostly work.
- **Built-in agent mode.** The editor can plan, edit multiple files, and run terminal commands without leaving the workspace.
- **Multi-model chat.** Switch between providers or run local Ollama models from the same chat panel.
- **Transparent pricing.** Paid tier covers managed inference; the editor itself is open source.
- **Community-driven roadmap.** Features land quickly and the issue tracker is active.

## Honest limitations

- **Newer project.** Less mature than Cursor or GitHub Copilot; some rough edges remain.
- **Agent depth still growing.** Long-horizon tasks are improving but not yet on par with Claude Code.
- **Self-hosting requires setup.** To use your own keys or local models, you need to configure providers.
- **Smaller ecosystem.** Fewer third-party tutorials and integrations than established editors.

## Best fit

Developers who want an AI-first editing experience, value open-source transparency, and are willing to trade some polish for control and auditability.
